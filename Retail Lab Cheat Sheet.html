Precision First

Good Habit

Q - What is the better default habit when I write a query?

Use SELECT with named columns by default. Keep SELECT * for prompts that explicitly ask for every column, or for quick table checks while practising.

Calculated Columns

Calculated Columns

Q - Show sale_id and calculate units_sold * unit_price. Name the calculated column revenue.

SELECT sale_id, units_sold * unit_price AS revenue FROM sales;
Calculated Columns

Q - Show sale_id and calculate (units_sold * unit_price) * (discount_pct / 100). Name the calculated column discount_amount.

SELECT sale_id, (units_sold * unit_price) * (discount_pct / 100.0) AS discount_amount FROM sales;
Calculated Columns

Q - Show sale_id and calculate units_sold - returned_units. Name the calculated column net_sold_units.

SELECT sale_id, units_sold - returned_units AS net_sold_units FROM sales;

Group And Summarise

Group And Summarise

Q - Display department and count its rows. Name the count column row_count.

SELECT department, COUNT(*) AS row_count FROM sales GROUP BY department;
Group And Summarise

Q - Display region and total units sold by region. Name the total column total_units_sold.

SELECT region, SUM(units_sold) AS total_units_sold FROM sales GROUP BY region;
Group And Summarise

Q - Display channel and average unit price by channel. Name the average column average_unit_price.

SELECT channel, AVG(unit_price) AS average_unit_price FROM sales GROUP BY channel;
Group And Summarise

Q - Display staff_name and total revenue by staff member. Name the total column total_revenue.

SELECT staff_name, SUM(units_sold * unit_price) AS total_revenue FROM sales GROUP BY staff_name;

Having

Having

Q - Display region and total units sold, named total_units_sold, but show only regions whose total is above 40.

SELECT region, SUM(units_sold) AS total_units_sold FROM sales GROUP BY region HAVING SUM(units_sold) > 40;
Having

Q - Display department and average unit price, named average_unit_price, but show only departments whose average is above 90.

SELECT department, AVG(unit_price) AS average_unit_price FROM sales GROUP BY department HAVING AVG(unit_price) > 90;
Having

Q - Display staff_name and total revenue, named total_revenue, but show only staff members whose total is above 3000.

SELECT staff_name, SUM(units_sold * unit_price) AS total_revenue FROM sales GROUP BY staff_name HAVING SUM(units_sold * unit_price) > 3000;
Filter, Group, Having And Order

Q - For Online sales only, display region and total units sold, named total_units_sold. Keep only regions with at least 18 online units, highest total first.

SELECT region, SUM(units_sold) AS total_units_sold FROM sales WHERE channel = 'Online' GROUP BY region HAVING SUM(units_sold) >= 18 ORDER BY total_units_sold DESC;
