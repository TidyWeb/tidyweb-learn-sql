Workshop SQL Practice

This lab deliberately stops being read-only. The point here is to mutate a simple table, see the effect, and reset when you want a clean slate. That makes the lesson concrete without risking any real data.

What This Lab Is For

  • Add rows with INSERT INTO.
  • Change existing rows with UPDATE ... SET ... WHERE ....
  • Remove rows with DELETE, then create, alter, and drop safe practice tables.

Read This Like A Beginner

        
When you write changing queries, slow down:

        
    
          
  • For UPDATE and DELETE, decide the WHERE first.
    • 
          
  • After a change, run a SELECT to confirm what happened.
    • 
          
  • If you break the table, use Reset Database and start over.
