Retail SQL Practice

This invented retail dataset is built for the middle stretch of SQL: using one table that has enough columns to make grouping and calculation feel real without adding join complexity. You can work on summaries, aliases, and the difference between row filters and grouped-result filters.

What This Lab Is For

  • Build calculated columns such as revenue and discount value.
  • Group by region, department, channel, or staff member.
  • Filter grouped results with HAVING after the grouping step.

Practice Prompts

        

      

      

        

          Calculated Columns
          
Start by making new values from old columns:

          
    
            
  • Show sale_id and calculate units_sold * unit_price. Name the calculated column revenue.
    • 
            
  • Show sale_id and calculate (units_sold * unit_price) * (discount_pct / 100). Name the calculated column discount_amount.
    • 
            
  • Show sale_id and calculate units_sold - returned_units. Name the calculated column net_sold_units.
    • 
          

        

        

          Group And Summarise
          
Then summarise the table:

          
    
            
  • Display department and count its rows. Name the count column row_count.
    • 
            
  • Display region and total units sold by region. Name the total column total_units_sold.
    • 
            
  • Display channel and average unit price by channel. Name the average column average_unit_price.
    • 
            
  • Display staff_name and total revenue by staff member. Name the total column total_revenue.
    • 
          

        

        

          Having
          
Finally filter grouped answers:

          
    
            
  • Display region and total units sold, named total_units_sold, but show only regions whose total is above 40.
    • 
            
  • Display department and average unit price, named average_unit_price, but show only departments whose average is above 90.
    • 
            
  • Display staff_name and total revenue, named total_revenue, but show only staff members whose total is above 3000.
    • 
            
  • For Online sales only, display region and total units sold, named total_units_sold. Keep only regions with at least 18 online units, highest total first.
    •
