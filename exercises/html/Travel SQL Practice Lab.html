Travel SQL Practice

This invented travel-booking table is built for deeper single-table thinking. You can compare rows against summary values, answer “which trips are above the average?” questions, and practise the difference between one result set and two result sets stacked together.

What This Lab Is For

  • Use WHERE to choose rows before grouping, then use HAVING to choose groups.
  • Use a subquery when the outer query needs a value computed from the whole table.
  • Compare one row against a group or table average.
  • Use UNION to stack compatible answers into one result set.

Practice Prompts

        

      

      

        

          Subqueries
          
Use a subquery for table-wide comparisons:

          
    
            
  • Display traveller_name, destination, and package_cost for trips with a package cost above the overall average.
    • 
            
  • Display traveller_name, destination, and nights for trips with more nights than the average trip.
    • 
            
  • Display traveller_name, destination, and review_score for trips whose review score is below the table average.
    • 
          

        

        

          Grouped Thinking
          
Use the logical order: choose rows, group them, choose groups, then sort:

          
    
            
  • Display travel_style and average package cost by travel style. Name the average column average_package_cost.
    • 
            
  • Display destination and average review score. Name the average column average_review_score, and show only destinations above 8.5.
    • 
            
  • Display booking_channel and total traveller count. Name the total column total_travellers, and show only booking channels above 8.
    • 
            
  • First keep only Winter trips. Then display booking_channel and the sum of travellers, named total_travellers. Group by channel and sort the totals highest first.
    • 
            
  • First keep only Website bookings. Then display travel_style and average package cost, named average_package_cost. Keep only averages above 500, then sort highest first.
    • 
            
  • Keep trips with review scores of at least 8. Display destination and total travellers, named total_travellers. Keep totals of at least 2, sort totals high to low then destination A–Z, and return the first 3 groups.
    • 
          

        

        

          Union
          
Stack results into one answer:

          
    
            
  • Return traveller_name, destination, and season for all winter trips and all luxury trips in one result using UNION.
    • 
            
  • Return traveller_name, destination, and booking_channel for all trips to Rome and all trips booked through agents in one result.
    • 
            
  • Return traveller_name, destination, and season for the same winter-and-luxury query, but use UNION ALL and compare what changes.
    •
