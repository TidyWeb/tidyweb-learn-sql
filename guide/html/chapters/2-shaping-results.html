Chapter 2 · Output Control

Shaping Results

A correct result can still be badly shaped. It may repeat the same value, arrive in no useful order, or contain far more rows than the question asks for. This chapter teaches how to turn the right records into a deliberate, checkable answer.

01Remove repetitionUse DISTINCT when the returned combinations should be unique.
02Define orderUse ORDER BY with an explicit direction.
03Keep a sliceUse LIMIT only after deciding what “first” means.
04Move the windowUse OFFSET to skip a known number of sorted rows.

Concept 1 · Unique returned values

DISTINCT Removes Duplicate Result Rows

DISTINCT acts on the columns named in the SELECT list. It asks SQL to return each selected value—or each selected combination—only once. It does not delete duplicates from the stored table.

Command bridge
SELECT DISTINCT venue FROM performances;
SELECTBegins the returned column list.
DISTINCTCollapses repeated result rows into one copy.
venueThe column whose returned values are checked for repetition.

DISTINCT Applies To The Whole Selected Combination

SELECT DISTINCT venue, seats
FROM performances;

This does not mean “make only venue distinct”. It returns unique venue-and-seats pairs. Studio, 60 and Studio, 80 are different result rows, so both remain.

Concept 2 · Deliberate order

ORDER BY Defines What Comes First

Without ORDER BY, SQL does not promise a meaningful row order. The order you happen to see is not a ranking you should rely on. Sorting is part of the question whenever words such as highest, lowest, newest, oldest, first, last, A–Z, or top appear.

Command bridge
ORDER BY ticket_price DESC
ORDER BYIntroduces the sorting rule.
ticket_priceThe first column SQL compares when arranging rows.
DESCDescending: high to low, newest to oldest, or Z to A.
ASCAscending: low to high, oldest to newest, or A to Z. This is usually the default, but writing it makes intent visible.
Worked example · sorted result
SELECT performance, ticket_price
FROM performances
ORDER BY ticket_price DESC;

Sort Ties With A Second Rule

When two rows tie on the first sort column, a second sort column makes their order predictable. SQL uses the second rule only inside a tie on the first.

SELECT performance, ticket_price
FROM performances
ORDER BY ticket_price DESC, performance ASC;

Read it as: highest price first; when prices tie, put performance names in A–Z order.

Concept 3 · A controlled number of rows

LIMIT Trims The Current Order

LIMIT does not understand words such as “best” or “largest”. It simply returns the requested number of rows from the front of the result as currently ordered. Ranking must come first.

Command bridge
ORDER BY seats DESC LIMIT 3;
ORDER BY seats DESCDefines largest seating capacity as the front of the result.
LIMITSays that only a fixed number of rows should be returned.
3The maximum number of rows. No equals sign and no brackets.
Worked example · “largest three”
SELECT performance, seats
FROM performances
ORDER BY seats DESC
LIMIT 3;
performanceseats
City Lights240
Winter Tales240
Open Strings140

Because two performances tie at 240, add a second sort rule if their order matters.

The Most Important Shaping Mistake

Wrong: LIMIT 3 by itself for “the three largest”.

Why: that means only “whichever three rows currently come first”.

Correction: translate the ranking word into ORDER BY and its direction, then apply LIMIT.

Concept 4 · Move the result window

OFFSET Skips Rows Before Returning Any

Pagination means dividing an ordered result into pages or chunks. OFFSET says how many rows to pass over before LIMIT begins collecting rows. A stable ORDER BY is essential; otherwise the pages have no dependable sequence.

Worked example · the second pair
SELECT performance
FROM performances
ORDER BY performance ASC
LIMIT 2 OFFSET 2;

First arrange all names A–Z. Then skip rows 1 and 2. Finally return the next two rows: positions 3 and 4 in that ordered result.

The complete shaping sequence

Build In Question Order, Write In SQL Order

  1. Decide the columns and source: SELECT ... FROM ....
  2. Keep only qualifying rows: WHERE ..., if needed.
  3. Collapse repeated selected combinations: DISTINCT, if the question asks for unique results.
  4. Define the ranking or sequence: ORDER BY ... ASC/DESC.
  5. Keep or skip a slice: LIMIT ... OFFSET ....
SELECT DISTINCT column_name
FROM table_name
WHERE row_condition
ORDER BY column_name DESC
LIMIT number OFFSET number;

How To Debug A Shaping Result

SymptomLikely place to inspect
Repeated values remain.Are the entire selected rows truly duplicates? Did you select an extra column that makes them different?
The right rows are in the wrong sequence.Check the ORDER BY column and whether the direction should be ASC or DESC.
The result shows three rows, but not the highest three.The LIMIT works; the missing or incorrect part is the ranking before it.
Pages overlap or change order.Make the sort stable, usually by adding a unique tie-breaker as a final ORDER BY column.
