SQL begins with one deceptively simple ability: turn an ordinary question into a precise request to one table. This chapter takes that journey slowly—from understanding what a table contains, through choosing columns and filtering rows, to deciding which kind of condition the question actually needs.

First, Know What You Are Looking At

The stored table

  • A table stores related facts in a grid.
  • A row is one record—in this table, one book record.
  • A column is one kind of fact, such as title or copies.
  • A cell is one value at the meeting point of a row and a column.

The returned result

A query is the question you write in SQL. The database reads it and returns a new, temporary table called a result set.

The original table is not cut down when you use SELECT. You are choosing what appears in the answer, not deleting stored data.

Concept 1 · Choose columns

SELECT Means “Show”

Start by asking: Which facts do I want the result to show? The answer to that question becomes the column list after SELECT. A comma separates one requested column from the next.

Command bridge
SELECT title, copies FROM library_books;

Plain English: show the title and number of copies from the library books table.

SELECTStarts a reading query and names what should appear in the result.
title, copiesThe two requested columns. The comma separates them; it does not mean “and” as a condition.
FROMIntroduces the source table.
library_booksThe table name. It must match a table that actually exists.
;The normal end marker for the SQL statement—like a full stop.
Worked example · visible result 
SELECT title, copies
FROM library_books;
titlecopies

Local Walks0

Five stored rows still produce five result rows, but the result contains only the two columns requested.

*: All Columns

SELECT *
FROM library_books;

Here the star is a shortcut for every column. It is useful when first inspecting a small table. Once the question is clear, named columns make the answer more deliberate and easier to check.

Common Wrong Turn

Wrong idea: the star means “everything everywhere.”

Correction: it means every column from the table named after FROM. It does not mean every table, and it is not the text-pattern wildcard in this position.

Concept 2 · Keep rows

WHERE Is A Yes-or-No Gate

Choosing columns answers “which facts?” Filtering answers “which records?” SQL tests the WHERE condition against each row. If the test is true, that row stays in the result. If false, it is left out.

Command bridge
SELECT title, copies FROM library_books WHERE copies >= 2;
WHEREIntroduces the row test.
copiesThe column tested in each row.
>=Means greater than or equal to. The boundary value counts.
2The number each row is compared with. Numbers are written without quotes.
Worked example · row-by-row reasoning 
SELECT title, copies
FROM library_books
WHERE copies >= 2;

Choose The Test That Matches The Question

Exact value or comparison

=Exactly equal to
!=Not equal to
> / <Greater than / less than
>= / <=Include the boundary too
WHERE branch = 'North'
WHERE copies != 0
WHERE copies <= 2

Text and numbers are written differently

Text values use single quotes because SQL must distinguish a literal piece of text from a column or table name. Numbers stay bare so SQL can treat them numerically.

WHERE category = 'Fiction'
WHERE copies = 3

Both tests: AND

Every joined condition must be true for the row to stay.

WHERE category = 'Practical'
  AND copies > 0

Either test: OR

At least one of the joined conditions must be true.

WHERE branch = 'Central'
   OR branch = 'West'

When AND And OR Meet

Mixed conditions can be read in more than one way by a person. Parentheses make your intended grouping visible.

WHERE category = 'Fiction'
  AND (branch = 'Central' OR branch = 'West');

This means: the row must be Fiction, and its branch may be either Central or West. The parentheses turn the two branch choices into one grouped decision.

A Range: BETWEEN

BETWEEN 1 AND 3 includes 1, 2, and 3. Both end values count.

WHERE copies BETWEEN 1 AND 3

A Short List: IN

IN checks one value against a list. It is a clearer form of several exact-match choices joined with OR.

WHERE branch IN ('Central', 'West')
Concept 3 · Exact text or text pattern

= And LIKE Ask Different Questions

Use = when the whole text value is known exactly. Use LIKE when the shape of the text matters—such as starts with, ends with, or contains.

%Any run of characters, including no characters.
_Exactly one character.
WHERE title = 'Local Walks'   -- exact whole value
WHERE title LIKE 'The%'       -- starts with The
WHERE title LIKE '%Guide%'    -- contains Guide
Worked example · pattern result 
SELECT title
FROM library_books
WHERE title LIKE 'The%';
title
The Long Way Home
The Hidden Harbour

The percent sign has special wildcard meaning because it is inside a pattern used with LIKE. Writing title = 'The%' would instead look for the literal full value The%.

Build the query without guessing

The Three-Pass Method

  1. Output: underline the facts the question asks you to display. Those become the SELECT columns.
  2. Source: identify the table containing those facts. That becomes FROM.
  3. Row rule: identify what must be true for a record to stay. That becomes WHERE, with the correct exact, comparison, list, range, or pattern test.

Read the finished query back in clause-sized English. If your reading does not match the original question, repair the logic before running it.

Frequent Early Mistakes

What went wrongWhat to check
Text was written without quotes.Literal text values normally need single quotes: 'Central'.
A wildcard was used with =.Pattern-shaped question means LIKE; exact known value means =.
A comma was used between conditions.Commas separate selected columns. Conditions are joined with AND or OR.
The result has the right rows but wrong columns.Repair the SELECT list; do not change a correct WHERE rule.
The result has the right columns but wrong rows.Repair the WHERE logic; the output list may already be right.
