SQLZoo field guide · nesting made visible

A question
inside a question

A subquery is a complete SQL question placed inside another SQL question. Read the inside first, then let its result answer the comparison in the outer query.

The key idea

Parentheses show where the supporting question begins and ends.

OUTER QUERY asks the main question.

main value > ( INNER QUERY )

INNER QUERY supplies the value or list the outer query needs.

Read it inside out

Each layer remains a complete query.

01 · The world example

Outer query + inner queryone number returned inside
SELECT name
FROM world
WHERE continent = 'Europe'
  AND gdp / population > (
      SELECT gdp / population
      FROM world
      WHERE name = 'United Kingdom'
  );

Inside: the benchmark

The inner query finds the United Kingdom row and calculates its gdp / population. It returns one number: the UK’s per-capita GDP.

Outside: the main question

The outer query keeps European countries whose own gdp / population is greater than that returned number.

current country's gdp / population>( United Kingdom's gdp / population )

02 · Why SELECT, FROM, WHERE appear twice

SELECT

Each query decides what its own question should return.

FROM

Each query names the table it is reading. The same table can be used again.

WHERE

Each query can filter its own rows. The inner filter identifies the UK.

03 · Parentheses hold different things

One returned value

Use a subquery inside a comparison when it returns one number or one value.

gdp / population > (
    SELECT gdp / population
    ...
)

A returned list

Use a subquery with IN or NOT IN when it returns several values.

yr NOT IN (
    SELECT yr
    FROM nobel
    ...
)

04 · Build it without getting jammed

1 · Outer questionWrite what the final result should show.
2 · Needed resultAsk what value or list the outer query needs.
3 · Inner queryWrite a complete SELECT, FROM, WHERE question.
4 · ParenthesesPut the complete inner query inside them.
5 · Type checkMake sure it returns a number, text value, or list.
6 · Read inside outTest the inner question mentally before the outer one.
Remember

Line breaks are not required, but they make nesting visible. More levels are possible, yet one clear supporting question at a time is the easiest way to understand the result.
