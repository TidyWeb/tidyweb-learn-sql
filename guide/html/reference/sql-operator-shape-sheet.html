SQLZoo field guide · syntax at a glance

The SQL operator
shape sheet

Read the shape before you write the query. The marks around a value tell you what kind of thing SQL is receiving: one value, a list, a pattern, a range, or an input to a function.

01 · The four things you must recognise

Identifier · column or table winner

Structural name. No quotes needed for a simple name.

Text value · single quotes 'Medicine'

Words stored in a cell. Quotes tell SQL it is text.

Number · no quotes 1984

Integer or decimal value. Treat it as a number.

List · parentheses ('a', 'b')

Several separate values, usually for IN.

Compare & combine

These usually take a column and a value or condition directly.

=

exactly equal to · subject = 'Peace'

<>  or  !=

not equal to · subject <> 'Peace'

<   ≤   >   ≥

compare numbers · strict or inclusive boundary

BETWEEN a AND b

inclusive range · no parentheses around the bounds

LIKE 'Sir%'

text pattern · % means any run of characters

NOT LIKE 'Sir%'

pattern does not match

IS NULL

missing value · not = NULL

AND · OR · NOT

join or reverse conditions · group with parentheses when needed

Lists, functions & order

These shapes have a special job: accept a list, calculate, or arrange results.

IN ('a', 'b')

one of these values · parentheses hold the list

NOT IN ('a', 'b')

none of these values

COUNT(column)

how many · parentheses hold the function input

COUNT(DISTINCT yr)

how many unique values

SELECT DISTINCT yr

show each value once · DISTINCT is not a function

ORDER BY yr DESC

sort newest first · ASC is the usual default

ORDER BY yr DESC, winner

two sort keys · comma means “then sort ties by…”

LIMIT 30

keep only a number of rows

02 · The parentheses test

Function input

COUNT( subject )

Parentheses hold what the function operates on.

List of values

IN ( 'a', 'b' )

Parentheses hold several comma-separated values.

Range

BETWEEN 1 AND 10

No wrapper around the two boundary values.

Text pattern

LIKE 'Sir%'

Quotes hold the text and its wildcard.

Logic grouping

( A AND B )

Parentheses group whole conditions.

Fast translation

“How many?” → COUNT   “starts with” → LIKE 'word%'   “one of” → IN (...)   “before” → <   “including” → <= or inclusive BETWEEN.

Keep the values and the structure distinct. Text → quotes · numbers → no quotes · lists/functions → parentheses
