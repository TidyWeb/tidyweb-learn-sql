Reference 2

Common SQL Mistakes

This page exists for exactly the places beginners tend to trip: the points where the grammar looks almost right, but the thinking behind it is wrong.

1. Using = When The Question Is Really A Pattern

The learner sees text and reaches for exact comparison even though the task is about shape.

Wrong

WHERE title = 'WALL-%'

Right

WHERE title LIKE 'WALL-%'

Rule: = means exact value. LIKE means pattern.

2. Reaching For NOT When != Is Simpler

This happens when the learner thinks “I want not,” but the actual job is just exact-value exclusion.

Overcomplicated Thought

NOT ... ?

Direct Form

WHERE club != 'Liverpool'

Rule: use != for “not this exact value.” Use NOT LIKE, NOT IN, or IS NOT NULL when you are reversing a broader condition.

3. Forgetting Quotes Around Text

Wrong

WHERE director = Christopher Nolan

Right

WHERE director = 'Christopher Nolan'

Rule: text values need quotes. Numbers do not.

4. Thinking LIMIT Means “Top” By Itself

Wrong Instinct

SELECT title
FROM films
LIMIT 5;

Right Move

SELECT title, imdb_rating
FROM films
ORDER BY imdb_rating DESC
LIMIT 5;

Rule: sort first if you mean top, bottom, newest, oldest, largest, or smallest.

5. Using = NULL

Wrong

WHERE distributor_id = NULL

Right

WHERE distributor_id IS NULL

NULL is not an ordinary value. Treat it as missingness, not as a text or numeric value to compare normally.

6. Losing Rows By Using The Wrong Join

The join itself works, but the result is thinner than expected because non-matching rows vanished.

Wrong For “Keep All Films”

FROM films
INNER JOIN boxoffice
ON films.film_id = boxoffice.film_id

Right For “Keep All Films”

FROM films
LEFT JOIN boxoffice
ON films.film_id = boxoffice.film_id

Rule: decide first whether unmatched rows should disappear or stay visible.

7. Using WHERE For Group Filters

Wrong

SELECT director, COUNT(*)
FROM films
WHERE COUNT(*) > 1
GROUP BY director

Right

SELECT director, COUNT(*)
FROM films
GROUP BY director
HAVING COUNT(*) > 1

Rule: WHERE filters rows before grouping. HAVING filters groups after grouping.

8. Forgetting What One Row Represents

This is less a typing mistake and more a thinking mistake.

Before you run a query, ask: should one row in my result mean one record, or one grouped category? If you cannot answer that, grouping mistakes are waiting for you.
